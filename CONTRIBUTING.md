# Contributing to Code Smasher

Thank you for considering contributing to Code Smasher! We welcome contributions from everyone. Here are the guidelines to follow:

## How to Contribute

1. **Fork the Repository**: Create your own fork of the repository.
2. **Create a Branch**: Create a new branch for your feature or bug fix.
3. **Make Changes**: Implement your changes in the forked repository.
4. **Commit Changes**: Write clear, concise commit messages.
5. **Push Changes**: Push your changes to your fork.
6. **Open a Pull Request**: Open a pull request to the main repository, detailing the changes you've made.

## Code of Conduct

- Be respectful and constructive in your comments.
- Use inclusive language.
- If you see someone struggling to contribute, offer assistance.

## Testing

- Ensure that your code is tested before submitting.
- Write new tests for new features. 

## Documentation

- Update documentation as needed.
- Contributions to the documentation are always welcome.

Thank you for contributing! Your help is greatly appreciated!
